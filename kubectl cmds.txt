when you install a new jump server . 
Steps to connect to the aks cluster :

install kubectl -- https://kubernetes.io/docs/tasks/tools/install-kubectl-linux/  ( use Debian steps for kubectl installation )
after installation verify using kubectl version

install helm --  https://helm.sh/docs/intro/install/  
make sure you have you helm-charts folder in your home directory
after installation verify using helm list

List the resources

kubectl get ns
kubectl get deployment
kubectl get pods
kubectl get svc
kubectl get rs
kubectl get hpa
kubectl get nodes
kuebctl get secrets
kubectl get ingress

Describe the resources 

kubectl describe deployment <depname>
kubectl describe pod <podname>
kubectl describe svc <svcname>
kubectl describe rs <rsname>
kubectl describe hpa <hpaname>
kubectl describe node <nodename>
kubectl describe ingress <ingressname>
kubectl describe secret <secretname>

Edit the resources 

kubectl edit deployment <depname>
kubectl edit pod <podname>
kubectl edit svc <svcname>
kubectl edit rs <rsname>
kubectl edit hpa <hpaname>
kubectl edit ingress <ingressname

Get logs from pods

kubectl logs <podname> <containername> >> filename.txt

Get /access resources from all namespace :

eg : kubectl get deployment -A

Get / access resources from other namespace :

eg : kubectl get deployment -n <namespacename>

Enter into the pod :

kubectl exec -it <podname> -- bash

To get all info like version namespace of any resource :

eg : kubectl get deployment -o wide

Delete any resource except deployment :

kubectl delete pod <podname>

Delete the deployment

 helm uninstall <deploymentname>

Helm Commands :

helm list
helm search repo       
helm search repo <depname>
helm show
helm show all
helm repo update
helm status <depname>
helm search hub
helm upgrade --install akv-secret-inject-0-0-local-deployment akv-secret-inject -f akv-secret-inject/env_configs/defaults.yaml -f akv-secret-inject/env_configs/nprd/defaults.yaml --namespace extensions -- Use this when u want to upgrade or when the particular deployment is missing 



Get cluster info :  kubectl cluster-info
Get version :  kubectl version

Command to take packet dumps :

tcpdump -ni eth0 -w /tmp/tsnr-20220427-capture01.pcap

Change into other namespace :

kubectl config set-context --current --namespace=default

how  to uninstall and reinstall nginx controller :

helm get values nginx-ingress-controller > values.yaml
helm uninstall nginx-ingress-controller
kubectl annotate $KIND $NAME meta.helm.sh/release-name=$RELEASE
kubectl annotate $KIND $NAME meta.helm.sh/release-namespace=$NAMESPACE
kubectl label $KIND $NAME app.kubernetes.io/managed-by=Helm
helm install nginx-ingress-controller ingress-nginx/ingress-nginx -f ./values.yaml
